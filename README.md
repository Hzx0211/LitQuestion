# LitQuestion

LitQuestion 是一个本地桌面 LLM 客户端。它不是想把聊天做得更花，而是想把一件很具体的小事做顺手：

当 GPT 给出一段我很喜欢、很关键的回答时，我经常会冒出一些很小的问题。
比如某个专业名词不懂，某个引用想确认，某个细节想顺手追问一下。

这些小问题如果直接塞回主聊天，会有两个问题：

- 主上下文会被越聊越杂，原本那条关键回答被很多枝节问题稀释掉
- 过一段时间再回头找那条真正有价值的回答，会很痛苦

所以 LitQuestion 做了一件事：把“小问题”从主对话里拎出来。

主窗口继续保留你的主线思路，副窗口专门承接那些轻量追问、概念补丁和临时分支。

> 它有点像上课时听 GPT 老师讲课。
> 途中遇到一个小问题，我不会举手打断老师，而是转头小声问同桌。
> 这样既不打乱老师的讲课节奏和主笔记，我也能马上把不懂的地方补上。

这就是 LitQuestion 的核心体验。

## 它解决什么问题

- 保住主对话的“干净上下文”
- 让关键回答不被零碎追问冲淡
- 让“临时查缺补漏”有地方放
- 让你回看时，主线和支线都更清楚

## 核心功能

- 主聊天窗口：承载主线问题和主要思路
- 副窗口追问：基于某条 AI 回答开出一条轻量分支
- 选择性并入主上下文：某些副链如果真的重要，再手动并回主线
- 高光概览：把你觉得最值得保留的 AI 回答单独标出来，方便回看
- 本地持久化：会话、消息、设置都保存在本地
- 自定义 OpenAI 兼容接口：支持自定义 Base URL、Model、API Key

## 适合谁

- 经常把 GPT 当老师、研究助手、写作搭子的人
- 会反复回看历史回答的人
- 很在意上下文干净度的人
- 讨厌“一个小问题把整段主线聊歪”的人

## 技术栈

- Tauri v2
- React 18 + TypeScript
- Vite 5
- Zustand
- SQLite

## 本地运行

### 1. 安装依赖

```bash
npm install
```

### 2. 启动开发环境

```bash
npm run tauri:dev
```

### 3. 构建桌面应用

```bash
npm run tauri:build
```

## 配置要求

LitQuestion 本身不提供模型服务，需要你自己提供兼容 OpenAI API 的模型接口信息：

- Base URL
- Model
- API Key

应用启动后可在设置页填写。

## Windows 安装包

仓库已经配置了 GitHub Actions 的 Windows 打包流程。

如果你只是想下载可分发的 Windows 安装包：

1. 打开仓库的 `Actions`
2. 进入 `Build Windows Installer`
3. 下载生成的 artifact

产物通常是一个类似下面的文件：

```text
LitQuestion_0.1.0_x64-setup.exe
```

## 为什么叫 LitQuestion

因为它不是要把所有问题都聊成一锅粥。

它更像是把主问题照亮，然后把那些轻一点、小一点、但又真实会冒出来的问题，放到旁边去处理。

主线继续往前走，小问题也不被忽略。
